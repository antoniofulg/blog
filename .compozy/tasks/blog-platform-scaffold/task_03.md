# Task 3: Drizzle Schema and Database Client

## Overview

Define the Drizzle ORM schema for the `posts` table, create the `postgres.js`-backed database client, configure Drizzle Kit for migration management, and run the initial migration to create the table. This is the core data layer that all backend tasks — the indexer (task_05), seed script (task_04), Better Auth adapter (task_10), and server functions in public and admin routes — depend on.

<requirements>
- MUST use `postgres.js` (not `pg`) as the Drizzle driver for Bun compatibility (ADR-001)
- MUST define the `posts` table with all columns specified in the TechSpec "Data Models" section: `id`, `file_path`, `slug`, `title`, `description`, `published_at`, `is_published`, `view_count`, `indexed_at`
- MUST use `file_path` as the unique stable identifier with a UNIQUE constraint (ADR-002)
- MUST use `slug` with a separate UNIQUE constraint (used for public routing)
- MUST configure `drizzle.config.ts` to point to the `drizzle/` directory for migration output
- MUST add `db:generate` (runs `drizzle-kit generate`) and `db:migrate` (runs `drizzle-kit migrate`) scripts to `package.json`
- MUST register `app/db/client.ts` in `vite-env-only` config in `app.config.ts` to prevent client-side bundling
- SHOULD export `Post` and `NewPost` TypeScript types inferred from the schema
</requirements>
