# ADR-005: Unified $slug loader resolves posts + static pages, posts win on collision

## Status

Accepted

## Date

2026-05-21

## Context

PRD F2 requires that adding a new static page (e.g., `app/content/pages/en/uses.mdx`) be a content-only change — no route file edit, no loader edit. Today the only dynamic route in `app/routes/{-$locale}/` is `$slug.tsx` (post lookup), and the only static page is `/about` served by a dedicated `about.tsx` route. The codebase has no catch-all or splat routes (TechSpec codebase scan confirmed).

Three structural options exist:

1. **Unified `$slug.tsx`** — the existing dynamic route handles both posts and static pages: try post lookup first, fall back to static page lookup, 404 if neither. URLs stay clean (`/uses`, `/about`, `/my-post`).
2. **Per-page route files** — keep `about.tsx`, add `uses.tsx`, `now.tsx`, etc. Each new page requires creating a route file — violates F2.
3. **Page-prefix route `/pages/$slug.tsx`** — pages live at `/pages/about`, `/pages/uses`. No collision possible; URL becomes noisier and asymmetric with posts.

Option 2 directly contradicts F2. Option 3 breaks existing `/about` URL (which currently resolves at root) and introduces a structural URL prefix the PRD never asked for.

A consequence of option 1 is that posts and pages share a slug namespace — a hypothetical post named `about` would collide with the existing `/about` static page. Three collision policies were considered:

- **Posts win** (matches existing `$slug.tsx` semantics; pages are silently shadowed).
- **Pages win** (structural-content priority; posts get shadowed).
- **Refuse to boot** (indexer detects collision at dev/build time and exits).

The user selected "posts win on collision" during TechSpec clarification.

## Decision

`app/routes/{-$locale}/$slug.tsx` becomes the single resolution surface for both posts and static pages. The route's server fn loader runs in this order:

1. Look up `posts` table for a row whose slug matches `params.slug` and whose file exists at `app/content/posts/<locale>/<slug>.mdx`.
2. If the post lookup fails, call `loadStaticPage(params.slug, params.locale)` from `app/lib/mdx/pages.server.ts` (ADR-001's encapsulated module).
3. If both lookups fail, throw `notFound()`.

On a slug collision (a post and a page sharing the same slug at the same locale), the **post wins** — the page is never reached. The collision is silent at runtime; the content-audit pipeline gains a warning-severity check that flags collisions during the next audit run.

The existing `app/routes/{-$locale}/about.tsx` route and its `about.server.ts` companion are deleted. `app/content/<locale>/about.mdx` moves into `app/content/pages/<locale>/about.mdx`.

## Alternatives Considered

### Alternative 1: Per-page route files (one `*.tsx` per page)

- **Description**: Keep the current pattern — every page is a hand-written route file (`about.tsx`, `uses.tsx`, `now.tsx`).
- **Pros**: Each page is independently typed and discoverable; clear file→URL mapping.
- **Cons**: Violates PRD F2 — adding a page requires a route file. Per-page loader copy-paste scales linearly with effort.
- **Why rejected**: Directly contradicts the PRD success criterion.

### Alternative 2: Page-prefix route `/pages/$slug.tsx`

- **Description**: Static pages live under `/pages/`. Example URLs: `/pages/about`, `/pages/uses`.
- **Pros**: No collision with post slugs; clear namespace.
- **Cons**: Breaks the existing `/about` URL (would become `/pages/about`); URL asymmetry with posts (`/my-post` vs `/pages/uses`); PRD never asked for it.
- **Why rejected**: URL break is a real regression for any existing inbound links to `/about` and adds nothing the user asked for.

### Alternative 3: Collision policy = "refuse to boot"

- **Description**: The indexer scans both `app/content/posts/<locale>/` and `app/content/pages/<locale>/` at startup; if any slug appears in both for the same locale, the process exits with a clear error.
- **Pros**: Footgun caught at the earliest possible point; impossible to silently shadow a page with a post.
- **Cons**: A typo in a post slug bricks dev/build; refusing to boot is a heavy hammer for a corpus that will probably never collide; the audit-time warning provides similar protection at lower cost.
- **Why rejected**: Asymmetric cost vs likelihood. Single-author, single-digit page count, and audit pipeline already exists. The hard-fail is not worth the disruption.

## Consequences

### Positive

- F2 satisfied: dropping a new `.mdx` under `app/content/pages/<locale>/` makes the route live with zero route or loader edits.
- One dynamic-slug resolution surface — easier to reason about, easier to test, easier to extend (e.g., adding RSS or analytics tracking for both posts and pages without forking logic).
- The `about.tsx` route and `about.server.ts` server fn get deleted (small LOC reduction, removes the per-page-loader anti-pattern).

### Negative

- Slug namespace is shared between posts and pages. Authors must be aware that posts shadow pages on collision.
- The `$slug.tsx` loader becomes slightly more complex (one extra fallback lookup).

### Risks

- **Silent shadowing**: a post slug collision silently hides a static page. *Mitigation*: add `slug-collision` finding to content-audit (warning severity); the next audit run surfaces the collision in the per-run report and SUMMARY.md row.
- **Loader latency**: post lookup + page lookup on every dynamic-slug request. *Mitigation*: post lookup is a single Drizzle query already; page lookup is `existsSync` + file read inside `pages.server.ts`. Combined still well under the < 200 ms p95 sitemap-comparable budget. No measurable regression expected.

## Implementation Notes

- `app/routes/{-$locale}/$slug.tsx` loader: try `getPostBySlugWithLangFn` first, then `loadStaticPage`, then `notFound()`.
- Delete `app/routes/{-$locale}/about.tsx` and `app/routes/{-$locale}/about.server.ts`.
- Move `app/content/<locale>/about.mdx` → `app/content/pages/<locale>/about.mdx` (`git mv` preserves history).
- Add a `slug-collision` finding to `app/lib/content-audit/checks.server.ts` — for each locale, intersect post slugs with page slugs; emit one finding per collision.
