# ADR-002: Delivery Strategy — Atomic Single PR

**Date:** 2026-05-06
**Status:** Accepted

## Decision

Deliver all 8 features (SOLID refactor, types directory, interface→type migration, shared session utility, MDX directory split, AGENTS.md, domain rule files, security findings report) in a single atomic PR on branch `TASK-0004/code-org-security`.

## Context

Three delivery approaches were evaluated for this code quality and agent tooling initiative. The key decision factor was rule-code alignment: rules written after the code refactor will accurately reflect implemented patterns, while rules written before may need revision.

## Alternatives Considered

- **Two-stream (B):** Code PR first, docs PR second. Smaller PRs but rules risk drifting from actual patterns; two CI cycles.
- **Docs-first (C):** Write rules before code. Slowest delivery; rules likely need revision after refactor reveals edge cases; two-cycle workflow with stall risk.

## Consequences

- Single larger PR (~10 files) to self-review — acceptable for solo developer
- Rules match implemented patterns exactly — no post-merge corrections needed
- Security findings report produced with fresh context from the live code audit
- One CI run gates all changes; one merge delivers complete quality baseline
- Future PRs can immediately use the established rules and AGENTS.md
