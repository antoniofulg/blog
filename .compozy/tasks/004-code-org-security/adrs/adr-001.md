# ADR-001: Scope for Code Organization + Security Hardening Task

**Date:** 2026-05-06
**Status:** Accepted

## Decision

Bound V1 scope to:
1. Extract server functions from route files into `*.server.ts` co-located files
2. Create `app/lib/session.ts` as shared auth utility (deduplicates requireSession)
3. Create `app/types/` directory and deduplicate PostFrontmatter interface
4. Minimal split of `mdx.server.ts` (parsing vs rendering concern)
5. Create AGENTS.md (root) + CLAUDE.md symlink + `.agents/rules/` domain files
6. Security findings report only — fixes in separate security task

## Context

Six SOLID violations and five security issues found in audit. Full restructure of DB layer and env validation code changes are out of scope for V1 to keep the PR reviewable and non-breaking.

## Alternatives Considered

- **Full restructure including DB layer and env validation code:** Too large; risks breaking changes to indexer/watcher pipeline
- **Documentation only (no code refactor):** Misses the structural debt that compounds with every new feature
- **Security fixes inline:** User chose findings-report-only to keep security work in a focused, dedicated task

## Consequences

- New routes must follow extracted server fn pattern (enforced by AGENTS.md rule)
- Security vulns remain open until security task is completed — acceptable for personal blog with single admin user
- Future refactors of watcher.server.ts and full mdx.server.ts split deferred to V2
