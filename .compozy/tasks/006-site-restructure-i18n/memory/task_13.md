## Objective Snapshot

Replaced `app/routes/about.tsx` (hardcoded TSX) with MDX-per-locale About pages. All deliverables shipped.

## Important Decisions

- Used `gray-matter` directly in `about.server.ts` (not `parseFrontmatter`) — parseFrontmatter returns PostFrontmatter which lacks `locale`/`links` fields
- Used `z.enum(["en", "pt-br"])` (hardcoded) instead of `z.enum(LOCALES)` to avoid readonly array casting issues with Zod 4
- `renderMdx` returns ComponentType; used `renderToStaticMarkup(createElement(Content, {}))` to get HTML (same pattern as $slug.server.ts)
- Static imports in `about.server.ts` (not dynamic) for testability via vi.mock
- NAV_LABELS update: kept en About as `"/about"`, updated pt-br About to `"/pt-br/about"`
- `switchLang` in header.tsx updated with two new cases: `rest === "about"` and `pathname === "/about"` both navigate to `/{-$locale}/about`
- Photo asset: copied `public/logo192.png` → `public/about/avatar.png` (real PNG, replaces User icon placeholder). Referenced in MDX body as `![Antonio Fulgencio](/about/avatar.png)`
- `lintFrontmatter` in mdx.test.ts now excludes `about.mdx` (different schema from posts)

## Learnings

- TanStack Router Link accepts any string for `to` (not strictly typed). Verified: `make check` passes with `to="/blog"` even when no route is registered at that path.
- `routeTree.gen.ts` was updated manually since `bunx tsr generate` requires a config file not present in this project. The file is auto-generated by the TanStack Start vite plugin during dev server start.
- `vi.fn()` without type arguments is the correct Vitest 4.x pattern — `vi.fn<[args], return>()` was removed in v4.
- `aboutFrontmatterSchema.parse` on gray-matter's `data` output works correctly — gray-matter preserves YAML types, including arrays of objects.
- `mdx.test.ts` `lintFrontmatter` scans all `content/` MDX files. Added filter `!f.endsWith("/about.mdx")` to exclude About from post-schema lint.
